# Trionex Store — AI Context

## ما هو هذا المشروع؟
متجر إلكتروني عربي (RTL) كامل باسم **Trionex Store**، يعمل كملف HTML واحد (`index.html`). الموقع منشور على GitHub Pages والبيانات تتزامن عبر Firebase Realtime Database.

## التقنيات المستخدمة
- **HTML/CSS/JS** — ملف واحد (single-file app)
- **Firebase Realtime Database** — مزامنة حية بين جميع الأجهزة للمنتجات، الطلبات، المحادثات
- **localStorage** — تخزين احتياطي + بيانات العميل (CID, الملف الشخصي)
- **Font Awesome 6** — أيقونات
- **Google Fonts (Tajawal)** — خط عربي احترافي
- **GitHub Pages** — استضافة الموقع المباشر

## بنية الملف
- `index.html` — الملف الرئيسي الوحيد (كل شيء في ملف واحد)
- `shop-img/` — صور احتياطية للمنتجات (webp, png, jpg)
- `backend/` — (غير مستخدم حالياً)
- `AI_CONTEXT.md` — هذا الملف

## الميزات الأساسية

### المتجر
- 4 تصنيفات منتجات: حيوانات، نباتات، قير (أدوات)، فلوس
- 15 منتج افتراضي يُنشئ عند أول تشغيل
- بحث، فلترة حسب التصنيف، ترتيب (اسم، سعر، أحدث، مبيعات)
- بطاقات منتجات مربعة/شبه مربعة في شبكة 4 أعمدة
- رفع صور للمنتجات (Base64) من لوحة الإدارة

### سلة الشراء والطلبات
- سلة محلية مع增减
- تأكيد الطلب مع إدخال الاسم ووسيلة التواصل
- حفظ بيانات العميل (الاسم + وسيلة التواصل + المعرف) تلقائياً في localStorage
- استعادة الطلبات عبر CID (رقم العميل)

### لوحة الإدارة
- دخول: `sadlaibadmin` / `sadlaibadmin`
- لوحة تحكم: إحصائيات (منتجات، طلبات، عملاء، بانتظار الرد)
- إدارة المنتجات: إضافة، تعديل، حذف، تفعيل/إلغاء، تعيين "منتجات مميزة"
- إدارة الطلبات: تغيير الحالة (قيد الانتظار ← قيد التحضير ← مكتمل/ملغي)، حظر العملاء
- إدارة المشرفين: إضافة/حذف مشرفين (للمالك فقط)
- الدردشة مع العملاء
- الأرشيف: إنهاء المحادثة + حذف نهائي
- إضافة/حذف مشرفين جدد

### نظام العملات
- SAR, DZD (36.5), USD (0.27), EUR (0.24), AED (0.98)
- اختيار العملة من قائمة منسدلة في النافبار
- تحفظ آخر عملة في localStorage

### ترتيب الصفحة الرئيسية
1. **⭐ المنتجات المميزة** — يحددها المشرف من لوحة الإدارة (زر ☆/⭐ في جدول المنتجات)، 4 منتجات كحد أقصى
2. **🔥 الأكثر مبيعاً** — 4 منتجات الأعلى مبيعاً

### الدردشة
- محادثة مباشرة بين العميل والمشرف
- اسم المشرف مخفي (يظهر كـ "🛡️ Support" أو "🛡️ الإدارة")

## المتغيرات المهمة في الكود

### DB
الكائن الرئيسي للبيانات:
- `DB.users` — المشرفين
- `DB.products` — المنتجات
- `DB.orders` — الطلبات
- `DB.chats` — الرسائل
- `DB.settings` — الإعدادات (banned, pv, sa)

### دوال مهمة
- `DB.saveAll()` — يحفظ كل شيء في localStorage + Firebase مع `Promise.allSettled`
- `fbOn(path, callback)` — يستمع للتغييرات من Firebase
- `pCard(p)` — ينشئ HTML بطاقة المنتج
- `rHome()` — يرسم الصفحة الرئيسية
- `rShop()` — يرسم صفحة المتجر
- `rOrd()` — يرسم طلبات العميل
- `adDash()`, `adProds()`, `adOrds()` — دوال لوحة الإدارة

### متغيرات حالة
- `cu` — كائن المشرف المسجل حالياً (null إذا غير مسجل)
- `curCode` — رمز العملة الحالي (SAR, USD, إلخ)
- `myOrd` — مصفوفة IDs طلبات العميل الحالي
- `cR` — ID الغرفة الحالية في الدردشة
- `adCur` — التبويب الحالي في لوحة الإدارة

## الملاحظات للمطور القادم
1. كل شيء في ملف HTML واحد — التعديلات تتم في `index.html` فقط
2. الـ CSS في `<style>` والـ JS في `<script>` قبل closing `</body>`
3. Firebase config موجود في `FIREBASE_CONFIG` — لا تغيّره
4. البيانات الافتراضية (seed) تُنشئ إذا كان `localStorage` فارغاً
5. الصور تُرفع كـ Base64 وتُخزّن في `p.imgData`
6. العملات rates ثابتة (hardcoded) — ليست مباشرة من API
7. الموقع RTL بالكامل — أي محتوى جديد يجب أن يكون بالعربية
8. لا تستخدم مكتبات خارجية إضافية — كل شيء في الملف
9. التصميم داكن — حافظ على التناسق اللوني
